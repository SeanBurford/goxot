# RFC 1613: Cisco Systems X.25 over TCP (XOT)

This document describes the implementation of RFC 1613 in GoXOT. RFC 1613 specifies a method for encapsulating X.25 Packet Level packets over a TCP connection.

## Encapsulation Format

GoXOT implements the encapsulation format defined in **Section 4.1 of RFC 1613**. This format uses a 4-octet header preceding each X.25 PLP packet, consisting of two 2-octet fields in Network Byte Order (Big Endian).

1.  **Version Header (2 octets)**: Set to zero (`0x0000`) as specified in the RFC.
2.  **Length Header (2 octets)**: The length in octets of the following X.25 PLP packet.
3.  **X.25 Packet**: The raw X.25 PLP packet data.

### Implementation Note: Protocol Alignment
The use of a 4-octet header (Version + Length) is the official specification. Some legacy documentation or early drafts may have focused solely on the "2-byte length," but GoXOT adheres to the strict 4-octet structure including the version field to ensure full RFC 1613 compatibility.

### Buffer Management
Because TCP is a stream-oriented protocol, implementations must buffer data to read exactly the number of bytes specified in the length header to preserve X.25 packet boundaries.

## TCP Connectivity

*   **Default Port**: 1998.
*   **Connection Initiation**: Either side can initiate the TCP connection. In GoXOT, `xot-server` typically waits for connections, while `xot-gateway` or external Cisco routers initiate them.

## Handling of Virtual Circuits

RFC 1613 maps X.25 Logical Channel Identifiers (LCIs) across the TCP stream.

### LCI 0 (Reserved)
LCI 0 is reserved for control packets that affect the entire link layer, specifically:
*   **Restart Request / Confirm**: Used to initialize the packet layer.
*   **Diagnostic**: Used to report error conditions not specific to a single call.

### Switched Virtual Circuits (SVC)
SVCs are established using standard X.25 Call Request procedures. The LCI is selected by the calling DTE and must be respected by the called DTE for that session.

### Permanent Virtual Circuits (PVC)
While RFC 1613 primarily discusses SVCs, it supports PVCs by pre-negotiating LCI ranges between peers.

## Operational Procedures

### Initialization (Restart)
Upon establishing the TCP connection, it is recommended (though not always strictly enforced by all implementations) to send an X.25 Restart Request to ensure both sides are in the **r1 (Packet Layer Ready)** state.

### Collision Handling
If both sides attempt to establish a call using the same LCI simultaneously, the "DTE/DCE" relationship (configured or negotiated) determines which call takes precedence. In GoXOT, we avoid this by using distinct LCI ranges for different gateways where possible.

### Disconnection
Closing the TCP connection implicitly clears all active virtual circuits on that link.
